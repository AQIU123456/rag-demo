# 阶段 4：AI 问答增强 (RAG) 实现说明

## 一、阶段 4 概述

本阶段实现了**RAG (Retrieval-Augmented Generation) 检索增强生成**功能，让系统能够基于知识库内容进行智能问答，并提供引用来源标注。

### 核心特性

1. **基于文档的精准问答**：利用向量检索从知识库中查找相关内容
2. **LLM 智能生成**：结合检索结果，由大语言模型生成准确答案
3. **引用来源标注**：清晰展示答案的知识来源和相似度分数
4. **防幻觉机制**：要求 LLM 仅基于提供的知识库内容回答

## 二、已实现的功能

### 后端实现

#### 1. RagService 服务 (`/service/RagService.java`)

核心方法：
- `answerWithRag(String question, int maxResults)` - RAG 问答主方法
- `buildContext()` - 构建检索结果的上下文
- `buildSystemPrompt()` - 构建系统提示词（包含防幻觉指令）
- `generateAnswer()` - 调用 LLM 生成答案
- `extractCitations()` - 提取引用来源

内部类：
- `RagResponse` - RAG 问答响应对象
- `Citation` - 引用来源信息

#### 2. ChatController 扩展 (`/controller/ChatController.java`)

新增 API 接口：
```
POST /api/chat/rag
```

请求示例：
```json
{
  "question": "公司的请假流程是什么？",
  "maxResults": 5
}
```

响应示例：
```json
{
  "success": true,
  "answer": "根据公司规定，员工请假需提前 3 天申请...",
  "context": "以下是从知识库中检索到的相关信息：...",
  "citations": [
    {
      "content": "员工请假需提前 3 天申请...",
      "score": 0.89,
      "source": "doc_1"
    }
  ]
}
```

## 三、技术架构

### RAG 工作流程

```
用户提问
   ↓
1. 语义检索 → EmbeddingService.searchSimilar()
   ↓
2. 构建上下文 → 整合格式化的检索结果
   ↓
3. 构建提示词 → System Prompt + Context + Question
   ↓
4. LLM 生成 → ChatLanguageModel.generate()
   ↓
5. 提取引用 → 整理引用来源信息
   ↓
6. 返回响应 → 答案 + 引用列表
```

### 系统提示词设计

```
你是一个专业的企业知识库助手。请根据以下提供的知识库内容来回答用户的问题。
要求：
1. 只基于提供的知识库内容进行回答，不要编造信息
2. 如果知识库中没有相关信息，请明确告知用户
3. 回答要准确、简洁、专业
4. 在回答中引用相关内容的编号，例如 [1]、[2]
5. 如果有多条相关信息，请综合整理后回答

以下是从知识库中检索到的相关信息：
[1] 内容片段... (相似度：0.89)
[2] 内容片段... (相似度：0.76)
...

请根据以上知识库内容回答用户的问题。
```

## 四、API 接口文档

### POST /api/chat/rag

**描述**: 基于知识库的 RAG 智能问答

**请求参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| question | String | 是 | 用户问题 |
| maxResults | Integer | 否 | 最大检索结果数，默认 5 |

**响应字段**:
| 字段 | 类型 | 说明 |
|------|------|------|
| success | Boolean | 是否成功 |
| answer | String | AI 生成的答案 |
| context | String | 检索到的上下文内容 |
| citations | Array | 引用来源列表 |
| citations[].content | String | 引用内容 |
| citations[].score | Double | 相似度分数 |
| citations[].source | String | 来源标识（如 doc_1） |

**错误处理**:
- 400: 问题为空
- 500: RAG 处理失败

## 五、使用示例

### 1. 上传知识文档

首先在知识库中上传相关文档：
- 访问"知识库"标签页
- 点击"上传文档"
- 输入文档内容（如公司规章制度）

### 2. 进行 RAG 问答

切换到"📚 AI 问答"标签页：
- 在输入框中输入问题
- 或直接点击示例问题
- 查看 AI 回答和引用来源

### 3. 调整检索参数

- 修改"最大检索结果数"（1-10）
- 更多结果 = 更全面的上下文，但可能引入噪声
- 较少结果 = 更精准的匹配，但可能遗漏信息

## 七、注意事项

### 依赖条件

1. **OpenAI API Key**: 需要有效的 API Key 用于嵌入和聊天模型
2. **向量化数据**: 需要先上传文档并完成向量化处理
3. **内存向量存储**: 当前使用 InMemoryEmbeddingStore，重启会丢失数据

### 性能考虑

1. **响应时间**: RAG 涉及多次向量检索和 LLM 调用，响应时间约 2-5 秒
2. **Token 消耗**: 上下文内容会增加 Token 使用量
3. **并发限制**: 注意 LLM API 的速率限制

### 生产环境建议

1. **切换到持久化向量库**: 使用 Milvus/Pinecone 等
2. **添加缓存层**: 缓存常见问题答案
3. **异步处理**: 对长文档进行异步向量化
4. **监控日志**: 记录 RAG 调用和性能指标
